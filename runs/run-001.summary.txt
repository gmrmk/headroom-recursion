## 2. ESTABLISHED

1. **[KNOWN] Relativization barrier.** There exist oracles A, B such that P^A = NP^A and P^B ≠ NP^B (Baker, Gill, Solovay (1975)). Consequence: any proof technique whose correctness is preserved under relativization (goes through unchanged when all machines are given access to an arbitrary oracle) cannot resolve P vs NP, since such a proof would apply equally to the A- and B-worlds and contradict one of the two oracle results. This restricts the space of admissible techniques but says nothing about which specific unconditional results are or are not relativizing.

2. **[KNOWN] Algebrization barrier.** Aaronson and Wigderson (2008) extend the oracle-separation technique to "algebrizing" proofs — those that remain valid when machines are given oracle access to a low-degree extension of a function over a larger field, in addition to the function itself. They exhibit algebrizing analogues of the BGS oracles, showing that arithmetization-based techniques (which underlie essentially all non-relativizing results known at the time, e.g., IP = PSPACE) also cannot, by themselves, separate P from NP. This is a strictly stronger barrier than relativization: a technique that evades relativization is not automatically safe unless it also evades algebrization.

3. **[KNOWN] Karp–Lipton theorem.** If NP ⊆ P/poly, then the polynomial hierarchy collapses to Σ2^p (Karp and Lipton (1980); the Σ2 form is due to Sipser, with a further strengthening by Yap). Proof idea (self-contained): assume NP ⊆ P/poly, so SAT has polynomial-size circuits. A Σ2^p predicate can existentially guess such a circuit C and then, using the downward self-reducibility of SAT, universally verify that C correctly decides SAT on all inputs of the relevant length — a guessed circuit that solves the search problem for satisfying assignments can be checked for correctness by a co-NP-style universal check. Given a correct polynomial-size SAT circuit available under a single existential guess, any Π2^p (equivalently any level-3) statement can be re-expressed with only two alternations, collapsing PH to Σ2^p. This is the standard tool linking nonuniform circuit assumptions to consequences for uniform complexity classes, and it is used in a load-bearing way in item 4.

4. **[KNOWN] Kannan's fixed-polynomial circuit lower bound.** For every constant k, there is a language in Σ2^p ∩ Π2^p that is not in SIZE(n^k) (Kannan (1982)). Proof (self-contained; the sole cited ingredient is the Lupanov upper bound, a standard textbook fact):

   *(i) A hard function trapped in the polynomial window, at log scale.* Fix k. Set **m = ⌈(k+1)·log₂ n⌉**, so n^{k+1} ≤ 2^m < 2n^{k+1}, i.e. 2^m = Θ(n^{k+1}). Work with Boolean functions on m inputs; the key structural fact is that a full m-input truth table has length 2^m = poly(n), so it is itself a polynomial-length object.

   - **(1a) Existence, by counting [proved here, elementary].** Over a fixed finite basis, the number of circuits of size ≤ n^k is at most 2^{O(n^k log n)}. The number of distinct m-input Boolean functions is 2^{2^m} ≥ 2^{n^{k+1}}. Since n^k log n = o(n^{k+1}), we have 2^{O(n^k log n)} < 2^{n^{k+1}} ≤ 2^{2^m} for all large n, so the evaluation map {circuits of size ≤ n^k} → {m-input functions} is not surjective: some m-input function g has circuit complexity C(g) > n^k. (This is the abstract pigeonhole step formalized in Lean Block 1.)

   - **(1b) Window membership, by Lupanov [cited ingredient, then elementary].** The Shannon–Lupanov upper bound states that *every* m-input Boolean function has circuit complexity ≤ (1+o(1))·2^m/m (Lupanov (1958); Arora–Barak §6.5). Applied to g:
     C(g) ≤ (1+o(1))·2^m/m ≤ (1+o(1))·2n^{k+1}/((k+1)·log₂ n) < n^{k+1} for all large n.
     Combined with (1a), C(g) ∈ (n^k, n^{k+1}]. The window is nonempty because its *width* n^{k+1} − n^k exceeds the Lupanov ceiling gap; no monotonicity-of-counts fact is invoked. (The Step 5 draft's "the number of realizable functions strictly increases between s=n^k and s=n^{k+1}" was circular — logically equivalent to the very claim "some function has complexity in (n^k, n^{k+1}]" — and is removed.)

   - **(1c) A definite hard function, and padding to n inputs.** Take g* = the lexicographically first m-input truth table with C(g*) > n^k (well-defined by (1a); it satisfies C(g*) ∈ (n^k, n^{k+1}] by (1b)). Pad g* to an n-input function ĝ that ignores its last n − m inputs. If ĝ had an n-input circuit of size ≤ n^k, fixing the ignored inputs to 0 yields a size-≤ n^k circuit for g*, contradicting C(g*) > n^k. Hence ĝ ∉ SIZE(n^k).

   *(ii) Definability in a fixed level of PH.* Because the truth table T of g* has poly length 2^m = Θ(n^{k+1}), "g* is the lex-first hard table" is a bounded-alternation predicate over poly-size objects. A single output bit ĝ(x) is captured by
     ∃T [ (∀ size-n^k circuit D, T differs from D on some input) ∧ (∀ T′ < T, ∃ size-n^k circuit computing T′) ∧ T_{x∣ₘ} = 1 (restriction of x to its first m bits) ],
   whose quantifier prefix (∃T over poly-length tables, then ∀T′ ∃circuit) sits at level Σ3^p; the internal "differs / computes" tests range over poly-length tables, poly-size circuits, and the 2^m = poly domain, hence are poly-time checkable. So the language of ĝ lies in Σ3^p. (Equivalently, one may guess a size-≤ n^{k+1} circuit directly; both witnesses are polynomial. The Step 2 error was guessing at the n-input scale, where the table has exponential length 2^n; at scale m the table is poly, so the guess is legitimate.)

   *(iii) Descent from Σ3^p to Σ2^p ∩ Π2^p by a Karp–Lipton win–win.* Split on whether NP is easy for size-n^k circuits.
   - **Case 1: NP ⊄ SIZE(n^k).** Then some language in NP ⊆ Σ2^p ∩ Π2^p already fails to have size-n^k circuits, and we are done directly.
   - **Case 2: NP ⊆ SIZE(n^k).** Then NP ⊆ P/poly, so by item 3 (Karp–Lipton) PH collapses to Σ2^p; consequently Σ3^p = Σ2^p = Π2^p, and the Σ3^p-definable hard language ĝ from (ii) lies in the now-flattened Σ2^p ∩ Π2^p, again giving a language in Σ2^p ∩ Π2^p not in SIZE(n^k).

   Since one of the two cases must hold, in every case Σ2^p ∩ Π2^p ⊄ SIZE(n^k), for every k. This is an unconditional circuit-size lower bound for a uniform class. Note what it does *not* do: it does not place a single fixed language outside P/poly (the witnessing language depends on k), and it is far short of separating P from NP. **Open point, not asserted as fact:** whether Kannan's argument relativizes is not verified here; the ESTABLISHED claim is only that the *statement* is strictly weaker than P ≠ NP or NP ⊄ P/poly, so its existence is not in tension with items 1–2 regardless of whether its proof relativizes.

   The propositional shape of the case split — "(Case 2 → collapse → conclusion) or (Case 1 → conclusion directly) ⟹ conclusion" — is pure logic independent of what NP, SIZE(n^k), etc. actually are, and is formalized abstractly in Lean Block 2 below.

```lean
import Mathlib

/-- Abstract pigeonhole fact underlying ingredient (1a): if there are strictly
fewer circuits of size ≤ n^k than there are Boolean functions on the input
domain (here modeled abstractly as a Fintype `Circuit` with an evaluation map
into functions `Dom → Bool`, where `Dom` stands for the m-input domain), then
some function is not computed by any such circuit. This captures exactly the
elementary counting step and nothing about where in the complexity spectrum the
resulting hard function lies — i.e. it proves (1a) only, not (1b). -/
theorem exists_hard_function
    {Circuit : Type*} [Fintype Circuit]
    {Dom : Type*} [Fintype Dom] [DecidableEq Dom]
    (eval : Circuit → (Dom → Bool))
    (h : Fintype.card Circuit < Fintype.card (Dom → Bool)) :
    ∃ f : Dom → Bool, ∀ c : Circuit, eval c ≠ f := by
  by_contra hcon
  push_neg at hcon
  have hsurj : Function.Surjective eval := hcon
  have hcard := Fintype.card_le_of_surjective eval hsurj
  omega
```

5. **[KNOWN] Natural proofs barrier.** Razborov and Rudich (1997) call a circuit-lower-bound strategy against a class C "natural" if it exhibits a combinatorial property of Boolean functions that is (a) *constructive*: decidable in time 2^{O(n)} given the 2^n-bit truth table of an n-input function; (b) *large*: satisfied by at least a 2^{-O(n)} fraction of all n-input Boolean functions (this is the correct threshold — the condition permits an exponentially small, but not doubly-exponentially small, fraction); and (c) *useful*: no function in C (for rich C, e.g. P/poly) satisfies the property for infinitely many n. They show essentially all then-known combinatorial lower-bound techniques (against AC0, AC0[p], formula size) are natural in this sense, and that if strong pseudorandom function generators exist (a standard cryptographic assumption), then no natural proof can establish superpolynomial lower bounds against P/poly — a large, constructive property would serve as a distinguisher breaking the pseudorandom family. Consequence for this attack line: any strategy extending Kannan-style counting/diagonalization into a genuine P/poly (or NP ⊄ P/poly) lower bound must either (i) avoid producing a "natural" property in the technical sense, or (ii) be conditioned on the failure of strong pseudorandom function generators. This is a barrier on lower-bound *methodology*, distinct from and additional to relativization/algebrization.

## 3. FRONTIER

**Next target.** Classify Kannan's Σ2^p ∩ Π2^p technique (item 4) against the three Razborov–Rudich conditions — with the correct logical shape, which is two-tier. "Which of constructivity, largeness, usefulness does Kannan's proof fail?" is *ill-posed*: it presupposes the technique exhibits a single combinatorial property of truth tables to test, and it does not. The corrected target is to establish (Tier 1) that no such property is exhibited, and (Tier 2) that the two canonical *forced* reads fail different conditions — so that "which condition fails" provably has no single answer.

**Tier 1 (category — the technique is not natural-shaped). [KNOWN].** Kannan's argument reaches its bound by counting (1a), the Lupanov window bound (1b), a lex-least definability step (ii), and a Karp–Lipton win–win case split (iii). At no point does it isolate a property Cₙ ⊆ {0,1}^{2^n} of truth tables that it then tests against P/poly; the case split even reasons about the *global* hypothesis "NP ⊆ SIZE(n^k)" rather than any per-function invariant. Hence the RR classification only becomes well-posed after one artificially reads a property out of the argument. This is folklore (RR 1997; Arora–Barak §22–23).

**Tier 2 (under-determination — the two forced reads fail different conditions). [KNOWN] throughout.**
- **Γ_A = {f : n-input, circuit complexity in (n^k, n^{k+1}]}** — the window the witness targets (after padding to n inputs). It fails **LARGENESS**: |Γ_A|/2^{2^n} ≈ 2^{−2^n(1−o(1))}, doubly-exponentially below RR's 2^{−O(n)} floor, because almost all functions have complexity ≈ Θ(2^n/n), far *above* the window (a one-line cardinality corollary of the Shannon–Lupanov count).
- **Γ_B = {f : circuit complexity > n^k}** — the property implicitly certified by "∉ SIZE(n^k)". It is **LARGE** (a 1 − 2^{−2^n(1−o(1))} fraction) and useful against SIZE(n^k), but its **CONSTRUCTIVITY** is not known: deciding C(f) > n^k from the 2^n-bit table is the MCSP-style minimum-circuit-size question, not known to lie in 2^{O(n)}. This is the only read that vindicates a "constructivity" focus — and only for Γ_B.
- **"useful against P/poly" is a category error, not a third read.** Γ_B is *satisfied* by many P/poly functions (any f with C(f) = n^{k+1} ∈ P/poly has C(f) > n^k), so it is not useful against P/poly at all. Kannan targets fixed-polynomial SIZE(n^k), strictly below RR's superpolynomial-P/poly scope. This scope mismatch — not a violated RR condition — is precisely why the natural-proofs barrier does not bite Kannan.

Since Γ_A fails largeness while Γ_B (at worst) fails constructivity, the "which condition fails" question has no single answer; that is the content of Tier 2.

(a) **Why it suffices (for progress, not resolution):** the two-tier picture converts the folklore "diagonalization/counting evades natural proofs by non-constructivity" into a precise account — (Tier 1) the technique exhibits no property to test, and (Tier 2) the only forced properties either fail largeness (Γ_A) or have unknown constructivity (Γ_B), while neither ranges over the superpolynomial-P/poly regime RR governs. This tells any would-be strengthener exactly what is missing: a property that is simultaneously large, constructive, *and* useful against superpolynomial P/poly — the exact combination RR shows is obstructed under strong PRFs.

(b) **Obstacle:** none of the Tier 1/Tier 2 observations is [NEW] — each is immediate from RR 1997 and the standard exposition (Arora–Barak §22–23); even the quantitative Γ_A fraction is a one-line cardinality corollary. They organize known barriers around a known theorem without clearing one. The deeper obstacle is structural: the attack line is anchored to a fixed-polynomial lower bound *below* the P/poly scope, so by construction it cannot produce a property in RR's governed regime, and no amount of classification yields P-vs-NP progress. Every "constructivity fails" observation here is [KNOWN] folklore (RR 1997), not [NEW].

(c) **Candidate strategy (honestly bounded):** the only route to genuine altitude from here leaves classification and targets a property that is large, constructive, and useful against *superpolynomial* P/poly while evading the PRF distinguisher — e.g., a property explicitly conditioned on the failure of strong PRFs, or a non-natural (non-constructive-by-design) certificate of hardness in the Williams algorithmic-method style (NEXP ⊄ ACC0, Williams (2011)) that certifies hardness without any 2^{O(n)}-time truth-table test.
