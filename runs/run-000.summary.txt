## 2. ESTABLISHED

1. **[KNOWN] Relativization barrier.** There exist oracles A, B such that P^A = NP^A and P^B ≠ NP^B (Baker, Gill, Solovay (1975)). Consequence: any proof technique whose correctness is preserved under relativization (goes through unchanged when all machines are given access to an arbitrary oracle) cannot resolve P vs NP, since such a proof would apply equally to the A- and B-worlds and contradict one of the two oracle results. This restricts the space of admissible techniques but says nothing about which specific unconditional results are or are not relativizing.

2. **[KNOWN] Algebrization barrier.** Aaronson and Wigderson (2008) extend the oracle-separation technique to "algebrizing" proofs — those that remain valid when machines are given oracle access to a low-degree extension of a function over a larger field, in addition to the function itself. They exhibit algebrizing analogues of the BGS oracles, showing that arithmetization-based techniques (which underlie essentially all non-relativizing results known at the time, e.g., IP = PSPACE) also cannot, by themselves, separate P from NP. This is a strictly stronger barrier than relativization: a technique that evades relativization is not automatically safe unless it also evades algebrization.

3. **[KNOWN] Karp–Lipton theorem.** If NP ⊆ P/poly, then the polynomial hierarchy collapses to Σ2^p (Karp and Lipton (1980); the Σ2^p form of the collapse is credited to Sipser in that paper). Proof (self-contained): assume NP ⊆ P/poly. By downward self-reducibility of SAT, the assumption yields polynomial-size *witness-producing* circuits: from a polynomial-size decision circuit for SAT one constructs, by the standard bit-by-bit self-reduction, a polynomial-size circuit that on a satisfiable formula outputs a satisfying assignment. Now let A ∈ Π2^p, say x ∈ A ⟺ ∀y ∃z φ(x, y, z) with φ polynomial-time and y, z polynomially bounded. The inner predicate ∃z φ(x, y, z) is an NP predicate, so it reduces to SAT. Claim: x ∈ A ⟺ ∃W ∀y [φ(x, y, W(x, y))], where W ranges over polynomial-size circuits and W(x, y) denotes the assignment W produces on the SAT instance encoding ∃z φ(x, y, z), decoded back to a candidate z. If x ∈ A, the assumed correct witness-producing circuit for the relevant input length serves as W, and it produces a valid z for every y. If x ∉ A, then some y₀ has no valid z at all, so *no* circuit W can make φ(x, y₀, W(x, y₀)) true, and the ∀y check fails for every W. The right-hand side is a Σ2^p predicate; hence **NP ⊆ P/poly ⇒ Π2^p ⊆ Σ2^p ⇒ PH = Σ2^p**. (For the last implication: Π2^p ⊆ Σ2^p gives, by complementation, Σ2^p = Π2^p; then Σ3^p = ∃·Π2^p ⊆ ∃·Σ2^p = Σ2^p, and induction flattens all higher levels.) This is the standard tool linking nonuniform circuit assumptions to uniform collapses, used in a load-bearing way in item 4. 4. **[KNOWN] Kannan's fixed-polynomial circuit lower bound.** For every constant k, there is a language in Σ2^p ∩ Π2^p that is not in SIZE(n^k) (Kannan (1982); statement confirmed by the retrieved snippet). Proof (self-contained, in two ingredients):

   *(i) A Σ3^p language of circuit complexity > n^k, via guessing a polynomial-size truth table at a scaled-down input length.* Fix k and set ℓ = ℓ(n) = ⌈(k+1) log₂ n⌉. The truth table of an ℓ-input Boolean function has 2^ℓ ≤ 2n^{k+1} bits — polynomial in n — so such a table is a legitimate existential witness. **Counting, at the level of functions:** there are 2^{2^ℓ} ≥ 2^{n^{k+1}} distinct Boolean functions on ℓ inputs, while each circuit of size ≤ n^k is described by O(n^k log n) bits, so circuits of size ≤ n^k compute at most 2^{O(n^k log n)} distinct functions. Since n^{k+1} eventually dominates any fixed multiple of n^k log n, for all sufficiently large n some ℓ-input function is computed by *no* circuit of size ≤ n^k. Define two predicates on ℓ-input truth tables t (for a given input length n):
   - **hard(t):** "no circuit D of size ≤ n^k computes t." For a fixed candidate D, "D computes t" is decidable in polynomial time: evaluate D on all 2^ℓ ≤ 2n^{k+1} inputs and compare with t's entries. So hard(t) = ∀D ¬[D computes t] is a Π1^p predicate.
   - **lexmin(t):** "every table t′ lexicographically smaller than t fails hard," i.e., ∀t′ < t ∃D of size ≤ n^k [D computes t′]. This is a Π2^p predicate (the innermost check is polynomial-time as above).

   For each sufficiently large n let t*ₙ be the lexicographically first hard table; it exists by the counting above and is unique, and it is the only table satisfying hard ∧ lexmin. Define the language L: for x with |x| = n, x ∈ L ⟺ ∃t [hard(t) ∧ lexmin(t) ∧ t(x₁…x_ℓ) = 1], where x₁…x_ℓ are the first ℓ bits of x (for the finitely many n with ℓ(n) > n or with no hard table, L^{=n} = ∅). The quantifier shape is ∃t [Π1^p ∧ Π2^p ∧ poly-time], with all quantified objects of polynomial length and a **free input x**, so L ∈ Σ3^p, and L^{=n} computes exactly x ↦ t*ₙ(x₁…x_ℓ).

   **Hardness of L (restriction bridge):** suppose for contradiction that for some sufficiently large n, L^{=n} is computed by a circuit C of size ≤ n^k. Fix C's last n − ℓ inputs to the constant 0 (fixing inputs to constants does not increase gate count). The resulting circuit C′ has ℓ inputs, size ≤ n^k, and computes x₁…x_ℓ ↦ t*ₙ(x₁…x_ℓ), i.e., C′ computes t*ₙ — contradicting hard(t*ₙ). Hence for all sufficiently large n, L^{=n} requires circuits of size > n^k; in particular L ∉ SIZE(n^k). (The finitely many small n are immaterial to the nonuniform lower bound.)

   *(ii) Descent from Σ3^p to Σ2^p ∩ Π2^p by a Karp–Lipton win–win.* Split on whether NP is easy for size-n^k circuits.
   - **Case 1: NP ⊄ SIZE(n^k).** Then some language in NP already fails to have size-n^k circuits, and NP ⊆ Σ2^p ∩ Π2^p (an NP predicate trivially pads to both a Σ2^p and a Π2^p form), so we are done directly.
   - **Case 2: NP ⊆ SIZE(n^k).** Then in particular NP ⊆ P/poly, so by item 3 the polynomial hierarchy collapses: PH = Σ2^p = Π2^p, hence Σ3^p = Σ2^p ∩ Π2^p. The language L from ingredient (i) then lies in Σ2^p ∩ Π2^p and is not in SIZE(n^k).

   Since one of the two cases must hold, in every case Σ2^p ∩ Π2^p ⊄ SIZE(n^k), for every k. Note carefully what this does *not* do: it does not place a single fixed language outside P/poly (the witnessing language depends on k), and it is far short of separating P from NP. **Open point, not asserted as fact:** whether Kannan's argument relativizes is not something this document verifies; the ESTABLISHED claim is only that the *statement* is strictly weaker than P ≠ NP or NP ⊄ P/poly, so its existence is not in tension with items 1–2 regardless of whether its proof relativizes — items 1–2 rule out relativizing/algebrizing proofs of statements that would separate P from NP (or place NP outside P/poly), not proofs of weaker statements like this one.

5. **[KNOWN] Natural proofs barrier.** Razborov and Rudich (1997) call a circuit-lower-bound strategy against a class C "natural" if it exhibits a combinatorial property of Boolean functions that is (a) *constructive*: decidable in time 2^{O(n)} given the 2^n-bit truth table of an n-input function (i.e., polynomial in the table length); (b) *large*: satisfied by at least a 2^{−O(n)} fraction of all n-input Boolean functions; and (c) *useful*: no function in C (for rich C, e.g. P/poly) satisfies the property for infinitely many n. They show essentially all then-known combinatorial lower-bound techniques (against AC0, AC0[p], formula size) are natural in this sense, and that if strong pseudorandom function generators exist (a standard cryptographic assumption), then no natural proof can establish superpolynomial lower bounds against P/poly — a large, constructive property would serve as a distinguisher breaking the pseudorandom family. Consequence for this attack line: any strategy extending Kannan-style counting/diagonalization into a genuine P/poly (or NP ⊄ P/poly) lower bound must either (i) avoid producing a "natural" property in the technical sense, or (ii) be conditioned on the failure of strong pseudorandom function generators. This is a barrier on lower-bound *methodology*, distinct from and additional to relativization/algebrization. (Citation not present in the provided snippets; flagged for mechanical audit.)

6. **[KNOWN] Circuit minimization and the status of constructivity.** The Minimum Circuit Size Problem (MCSP) — given the full truth table of a Boolean function and a size parameter s, decide whether the function has a circuit of size ≤ s — is in NP (a size-≤ s circuit is a short witness, verifiable by evaluating it against the whole table). Kabanets and Cai (2000) connect MCSP to the natural proofs framework: if MCSP were decidable in time polynomial in the table length, then the property "circuit complexity exceeds s" would become *constructive* in the sense of item 5, and — being also large and useful — it would break strong pseudorandom function generators via the Razborov–Rudich argument; hence, under that standard cryptographic assumption, MCSP ∉ P. Two guardrails on interpretation, both load-bearing for the FRONTIER: (i) this is a **conditional** result — unconditionally, it bounds *known algorithms only*; no unconditional lower bound for deciding hardness-of-truth-tables predicates is known; (ii) it shows the constructivity status of "hard(t)"-type properties (item 4(i)) is a recognized and studied question, so the observation that deciding such properties naively requires enumerating all small circuits is **anticipated** by Razborov and Rudich (1997) and Kabanets and Cai (2000) and cannot be claimed as [NEW] by this document. Scope hedge, stated now to prevent overclaiming later: Razborov–Rudich usefulness in item 5 targets superpolynomial/P-poly lower bounds, so the fixed-size property hard(t) engages the barrier only against pseudorandom function families computable in circuit size below n^k; a classification of hard(t) informs but does not by itself instantiate the full P/poly barrier.

(b) **Obstacle:** Constructivity demands a decision procedure for hard(t) running in time 2^{O(ℓ)} = poly(N).
