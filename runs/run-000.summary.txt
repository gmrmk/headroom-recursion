## 2. ESTABLISHED

1. **[KNOWN] Relativization barrier.** There exist oracles A, B such that P^A = NP^A and P^B ≠ NP^B (Baker, Gill, Solovay (1975)). Consequence: any proof technique whose correctness is preserved under relativization (goes through unchanged when all machines are given access to an arbitrary oracle) cannot resolve P vs NP, since such a proof would apply equally to the A- and B-worlds and contradict one of the two oracle results. This restricts the space of admissible techniques but says nothing about which specific unconditional results are or are not relativizing.

2. **[KNOWN] Algebrization barrier.** Aaronson and Wigderson (2008) extend the oracle-separation technique to "algebrizing" proofs — those that remain valid when machines are given oracle access to a low-degree extension of a function over a larger field, in addition to the function itself. They exhibit algebrizing analogues of the BGS oracles, showing that arithmetization-based techniques (which underlie essentially all non-relativizing results known at the time, e.g., IP = PSPACE) also cannot, by themselves, separate P from NP. This is a strictly stronger barrier than relativization: a technique that evades relativization is not automatically safe unless it also evades algebrization.

3. **[KNOWN] Karp–Lipton theorem.** If NP ⊆ P/poly, then the polynomial hierarchy collapses to Σ2^p (Karp and Lipton (1980); the Σ2 form is due to Sipser, with a further strengthening by Yap). Proof idea (self-contained): assume NP ⊆ P/poly, so SAT has polynomial-size circuits. A Σ2^p predicate can existentially guess such a circuit C and then, using the downward self-reducibility of SAT, universally verify that C correctly decides SAT on all inputs of the relevant length — a guessed circuit that solves the search problem for satisfying assignments can be checked for correctness by a co-NP-style universal check. Given a correct polynomial-size SAT circuit available under a single existential guess, any Π2^p (equivalently any level-3) statement can be re-expressed with only two alternations, collapsing PH to Σ2^p. This is the standard tool linking nonuniform circuit assumptions to consequences for uniform complexity classes, and it is used in a load-bearing way in item 4.

4. **[KNOWN] Kannan's fixed-polynomial circuit lower bound.** For every constant k, there is a language in Σ2^p ∩ Π2^p that is not in SIZE(n^k) (Kannan (1982)). Proof (self-contained, in two ingredients):

   *(i) A Σ3^p upper bound for a hard function via circuit-guessing, not table-guessing.* One cannot existentially guess the full truth table of a hard n-input function: that table is 2^n bits, exponential, so it is not a polynomial-length witness. The correct guessable witness is a *circuit*. Consider the predicate: "there exists a circuit C of size n^{k+1} such that for every circuit D of size n^k there exists an input x with C(x) ≠ D(x)." A circuit of size n^{k+1} is a polynomial-length object, hence a legitimate existential witness; the inner "for every D of size n^k there exists x" is a Π2^p condition on C. So the language computed by such a witnessed C is captured by a Σ3^p predicate of the form ∃C (size n^{k+1}) ∀D (size n^k) ∃x [C(x) ≠ D(x)]. Existence of a witness C is guaranteed by counting deployed *against circuits*: there are at most 2^{O(n^k log n)} circuits of size ≤ n^k, whereas there are 2^{O(n^{k+1} log n)} circuits of size n^{k+1} computing far more distinct functions; hence some size-n^{k+1} circuit computes a function differing from every size-n^k circuit. This places a language of circuit complexity in (n^k, n^{k+1}] inside Σ3^p, but Σ3^p is not yet the claimed bound.

   *(ii) Descent from Σ3^p to Σ2^p ∩ Π2^p by a Karp–Lipton win–win.* Split on whether NP is easy for size-n^k circuits.
   - **Case 1: NP ⊄ SIZE(n^k).** Then some language in NP ⊆ Σ2^p ∩ Π2^p already fails to have size-n^k circuits, and we are done directly.
   - **Case 2: NP ⊆ SIZE(n^k).** Then in particular NP ⊆ P/poly, so by item 3 (Karp–Lipton) the polynomial hierarchy collapses to Σ2^p; consequently Σ3^p = Σ2^p = Π2^p, and the Σ3^p-definable hard language from ingredient (i) lies in the now-flattened Σ2^p ∩ Π2^p, again giving a language in Σ2^p ∩ Π2^p not in SIZE(n^k).

   Since one of the two cases must hold, in every case Σ2^p ∩ Π2^p ⊄ SIZE(n^k), for every k. This is an unconditional circuit-size lower bound for a uniform class. Note carefully what it does *not* do: it does not place a single fixed language outside P/poly (the witnessing language depends on k), and it is far short of separating P from NP. **Open point, not asserted as fact:** whether Kannan's argument relativizes is not something this document verifies; the ESTABLISHED claim is only that the *statement* is strictly weaker than P ≠ NP or NP ⊄ P/poly, so its existence is not in tension with items 1–2 regardless of whether its proof relativizes — items 1–2 rule out relativizing/algebrizing proofs of statements that would separate P from NP (or place NP outside P/poly), not proofs of weaker statements like this one.

   *(Correction note relative to Step 2: the earlier draft claimed the bound followed from a Σ2^p machine that "existentially guesses a hard truth table"; that is invalid, since a 2^n-bit table is not a polynomial-length witness. The exponential-table obstruction is resolved above by guessing a circuit (giving Σ3^p) and then descending to Σ2^p ∩ Π2^p via the Karp–Lipton case split, which is the genuinely non-trivial ingredient.)*

5. **[KNOWN] Natural proofs barrier.** Razborov and Rudich (1997) call a circuit-lower-bound strategy against a class C "natural" if it exhibits a combinatorial property of Boolean functions that is (a) *constructive*: decidable in time 2^{O(n)} given the 2^n-bit truth table of an n-input function; (b) *large*: satisfied by at least an inverse-polynomial fraction of all n-input Boolean functions; and (c) *useful*: no function in C (for rich C, e.g. P/poly) satisfies the property for infinitely many n. They show essentially all then-known combinatorial lower-bound techniques (against AC0, AC0[p], formula size) are natural in this sense, and that if strong pseudorandom function generators exist (a standard cryptographic assumption), then no natural proof can establish superpolynomial lower bounds against P/poly — a large, constructive property would serve as a distinguisher breaking the pseudorandom family. Consequence for this attack line: any strategy extending Kannan-style counting/diagonalization into a genuine P/poly (or NP ⊄ P/poly) lower bound must either (i) avoid producing a "natural" property in the technical sense, or (ii) be conditioned on the failure of strong pseudorandom function generators. This is a barrier on lower-bound *methodology*, distinct from and additional to relativization/algebrization. (Citation not present in the provided snippets; flagged for mechanical audit.)

## 3. FRONTIER

(a) **Why it suffices (for progress, not resolution):** A rigorous classification of Kannan's technique against the natural-proofs criteria would convert the folklore "diagonalization evades natural proofs because it is non-constructive" into a checked, citable claim, and clarify exactly what a Σ2^p-style counting argument would need in order to reach an unconditional P/poly bound — sharpening the remaining obstacle rather than gesturing at "the barriers."

(b) **Obstacle:** "Constructive" is a statement about existence of a 2^{O(n)}-time predicate on truth tables, whereas Kannan's argument produces its language via existential quantification over circuits inside a Σ2^p/Σ3^p machine. Translating "solved by a Σ2^p machine" into "yields (or does not yield) a 2^{O(n)}-time-decidable large property of truth tables" requires care: bounded-alternation-verifiable properties are not obviously equivalent to, nor automatically excluded from, deterministic-2^{O(n)}-checkable ones. Doing this translation without hand-waving is nontrivial and has not been done here.
