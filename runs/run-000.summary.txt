## 2. ESTABLISHED

1. **[KNOWN] Relativization barrier.** There exist oracles A, B with P^A = NP^A and P^B ≠ NP^B (Baker, Gill, Solovay (1975)). Consequently any technique whose correctness is preserved under relativization (survives giving all machines an arbitrary oracle) cannot resolve P vs NP: it would apply verbatim to both worlds and contradict one. The Lean block below captures only the propositional shape: no single truth value for an abstract oracle-indexed predicate `PeqNP` is consistent with witnesses on both sides. It does not mechanize oracle machines or the meaning of "P^A = NP^A"; it earns type-correctness only. Hypotheses are passed as parameters rather than axioms.

   ```lean
   theorem relativization_barrier
       {Oracle : Type} (PeqNP : Oracle → Prop)
       (oracle_A oracle_B : Oracle)
       (BGS_A : PeqNP oracle_A) (BGS_B : ¬ PeqNP oracle_B) :
       ¬ (∀ o : Oracle, PeqNP o) ∧ ¬ (∀ o : Oracle, ¬ PeqNP o) :=
     ⟨fun h => BGS_B (h oracle_B), fun h => h oracle_A BGS_A⟩
   ```

2. **[KNOWN] Algebrization barrier.** Aaronson and Wigderson (2008) extend oracle separations to "algebrizing" proofs — valid even when machines get oracle access to a low-degree extension of a function over a larger field. They exhibit algebrizing analogues of the BGS oracles, so arithmetization-based methods (which underlie the non-relativizing results known then, e.g. IP = PSPACE) also cannot by themselves separate P from NP. This is a distinct, harder-to-evade barrier than item 1: evading relativization does not imply evading algebrization. No Lean block is given — the object-level content quantifies over low-degree extensions across extension fields, and a propositional skeleton would either trivialize the statement or reintroduce the theorem as an axiom.

3. **[KNOWN] Karp–Lipton theorem.** If NP ⊆ P/poly then PH collapses to Σ₂ᵖ (Karp and Lipton (1980); Σ₂ form due to Sipser). Self-contained idea: assume SAT has poly-size circuits. A Σ₂ᵖ predicate existentially guesses a candidate circuit family C and, using the downward self-reducibility of SAT, universally verifies that C decides SAT correctly on all inputs of the relevant length (a guessed search-circuit is checked by a co-NP universal check). With a certified poly-size SAT circuit obtainable under one existential guess, any Σ₃ᵖ statement re-expresses with two alternations, collapsing PH to Σ₂ᵖ. Load-bearing in item 4, Case 2. The Lean block records only that item 4 uses exactly this inference (modus ponens); `NPinPpoly`, `PHeqSigma2` are opaque propositions and the hypothesis is a parameter.

   ```lean
   theorem karp_lipton_applied
       {NPinPpoly PHeqSigma2 : Prop}
       (karp_lipton : NPinPpoly → PHeqSigma2)
       (h : NPinPpoly) : PHeqSigma2 :=
     karp_lipton h
   ```

4. **[KNOWN] Kannan's fixed-polynomial circuit lower bound.** For every constant k there is a language in Σ₂ᵖ ∩ Π₂ᵖ not in SIZE(nᵏ) (Kannan (1982); textbook treatment Arora–Barak (2009), Ch. 6). Proof in two ingredients.

   *(i) A Σ₃ᵖ upper bound for a well-defined hard function via few variables + lex-least truth table.* Fix ℓ := (k+2)·⌈log n⌉ input bits, so 2^ℓ = n^{k+2}. The number of Boolean functions on ℓ bits is 2^{2^ℓ} = 2^{n^{k+2}}, while the number of circuits of size ≤ nᵏ over a fixed fan-in-2 basis is at most 2^{O(nᵏ log n)}. Since n^{k+2} ≫ nᵏ log n for large n, counting (Shannon (1949)) forces some ℓ-variable function to have circuit complexity > nᵏ. The truth table of an ℓ-variable function is 2^ℓ = n^{k+2} bits — polynomial — so it is a legitimate existential witness, the domain being ℓ = Θ(log n) bits rather than n. Define T\* := the lexicographically least ℓ-bit truth table with no size-nᵏ circuit; uniqueness makes T\* well-defined. Its defining predicate:
   - "no size-nᵏ circuit computes T": ∀ D (size nᵏ), D ≢ T — for fixed D and T, checkable in poly time by evaluating D on all 2^ℓ = poly inputs, so this conjunct is Π₁;
   - "lex-least such": ∀ T′ < T, ∃ D (size nᵏ) with D ≡ T′ — this is Π₂.

   The conjunction "T = T\*" is Π₂. Indexing n-bit inputs x by their first ℓ bits, **x ∈ L iff ∃T [ (T = T\*) ∧ T(x)=1 ]**, an ∃-over-Π₂ predicate, i.e. **Σ₃ᵖ**. Hardness transfer (standard padding): a size-nᵏ circuit for L on n-bit inputs yields, by hardwiring the ignored bits, a size-nᵏ circuit for the ℓ-variable function T\*, contradicting its construction; hence L ∉ SIZE(nᵏ). (Lupanov (1958) guarantees every ℓ-function has some circuit of size O(2^ℓ/ℓ) = O(n^{k+2}/log n); this is a sanity remark, not load-bearing, since the table is guessed, not a circuit.)

   The counting existence step is mechanized abstractly below (pigeonhole on `compute : Circuit → BoolFn`); it captures that combinatorial core only, not the alternation bookkeeping.

   ```lean
   import Mathlib

   theorem exists_hard_function
       {Circuit BoolFn : Type} [Fintype Circuit] [Fintype BoolFn]
       (compute : Circuit → BoolFn)
       (h : Fintype.card Circuit < Fintype.card BoolFn) :
       ∃ f : BoolFn, ∀ c : Circuit, compute c ≠ f := by
     by_contra hcon
     push_neg at hcon
     have hsurj : Function.Surjective compute := fun f => hcon f
     have hle := Fintype.card_le_of_surjective compute hsurj
     omega
   ```

   *(ii) Descent from Σ₃ᵖ to Σ₂ᵖ ∩ Π₂ᵖ by a Karp–Lipton win–win.* Split on whether NP is easy for size-nᵏ circuits.
   - **Case 1: NP ⊄ SIZE(nᵏ).** Some NP language already lacks size-nᵏ circuits. And NP ⊆ Σ₂ᵖ ∩ Π₂ᵖ unconditionally: NP = Σ₁ᵖ ⊆ Σ₂ᵖ (a predicate ∃y φ equals ∃y ∀z φ with z over a one-point dummy domain), and NP = Σ₁ᵖ ⊆ Π₂ᵖ (equivalently ∀z ∃y φ over the same dummy z). So that language lies in Σ₂ᵖ ∩ Π₂ᵖ and is not in SIZE(nᵏ). Done.
   - **Case 2: NP ⊆ SIZE(nᵏ).** Then NP ⊆ P/poly, so by item 3 PH collapses to Σ₂ᵖ; hence Σ₃ᵖ = Σ₂ᵖ = Π₂ᵖ, and the Σ₃ᵖ-language L from (i) lies in the flattened Σ₂ᵖ ∩ Π₂ᵖ and is not in SIZE(nᵏ). Done.

   One case always holds, so Σ₂ᵖ ∩ Π₂ᵖ ⊄ SIZE(nᵏ) for every k. **Scope:** the witnessing language depends on k; this places no single language outside P/poly and is strictly weaker than P ≠ NP. Its existence is consistent with items 1–2 precisely because the statement is weaker than NP ⊄ P/poly, regardless of whether the proof relativizes.

5. **[KNOWN] Natural proofs barrier.** Razborov and Rudich (1997) call a lower-bound strategy against a class 𝒞 "natural" if it exhibits a property of Boolean functions that is (a) *constructive* — decidable in 2^{O(n)} time from the 2ⁿ-bit truth table; (b) *large* — holds for ≥ an inverse-poly fraction of all n-input functions; (c) *useful* — no function in 𝒞 satisfies it infinitely often. They show essentially all then-known combinatorial techniques (AC⁰, AC⁰[p], formula size) are natural, and that if strong pseudorandom function generators exist, no natural proof yields superpolynomial lower bounds against P/poly. Consequence: any extension of Kannan-style counting/diagonalization into a genuine NP ⊄ P/poly bound must either avoid producing a natural property or be conditioned on the failure of strong PRFGs.

The next result must therefore live in a **non-relativizing, non-natural** technique class. The concrete candidate lever: **hardness magnification** for a sparse / meta-computational problem.

**Precise target.** Let MCSP[s] be: given the 2ⁿ-bit truth table of an n-input function, decide whether it has a circuit of size ≤ s(n). A representative magnification theorem (Oliveira–Pich–Santhanam (2019); Chen–Jin–Williams (2020)) has the shape: *there is a threshold θ such that a lower bound of the form "MCSP[2^{√n}] (or a suitably parameterized gap-MKtP) requires circuits of size N·polylog(N) at a specific depth/locality" already implies NP ⊄ P/poly (or a fixed superpolynomial separation).*

(a) **Why it suffices.** Magnification converts a *barely superlinear* lower bound on one sparse problem into a *superpolynomial* separation — crossing exactly the P/poly line that items 1, 2, 5 block the Kannan line from reaching. The needed bound is quantitatively just above what current unconditional methods (which prove N^{1+ε}-type bounds against restricted models) achieve.

(b) **Obstacle.** The **locality barrier** (Chen–Jin–Williams (2020)): the same magnification theorems show that all *known* techniques capable of proving lower bounds in that regime are "local" in a precise sense, and local techniques provably *cannot* prove the magnifying lower bound. Magnification thus trades the relativization/natural-proofs barriers for a new, sharply characterized one — narrower and technique-specific, but not yet circumvented.
